# CLAUDE.md — AFFiNE AWS Demo

## Project overview

Two parallel approaches for self-hosting [AFFiNE](https://affine.pro) (open-source Notion alternative) on an AWS Free Tier EC2 instance:

- **v3 (current) — Terraform**: single `terraform apply` provisions everything; `terraform destroy` tears it down
- **v2 (legacy) — Shell scripts**: five sequential scripts in `scripts/`; still functional

## Repository layout

```
affine-aws-demo/
├── terraform/                   # v3 — Terraform root module (recommended)
│   ├── versions.tf              # Provider pins (aws ~>5, tls, random, http, local)
│   ├── variables.tf             # All inputs with defaults
│   ├── main.tf                  # VPC, IGW, subnet, route table, security group
│   ├── iam.tf                   # IAM group/user/policy/access-key
│   ├── s3.tf                    # S3 bucket + public access block
│   ├── ec2.tf                   # Key pair, random secrets, AMI lookup, EC2, EIP
│   ├── outputs.tf               # public_ip, access_url, ssh_command, etc.
│   ├── terraform.tfvars.example # Copy → terraform.tfvars to customise
│   └── templates/
│       └── user_data.sh.tpl     # Cloud-init: installs Docker/Nginx, deploys AFFiNE
├── scripts/                     # v2 — legacy shell-script deployment
│   ├── 01-aws-bootstrap.sh      # Create VPC, IAM, S3, key pair, Elastic IP
│   ├── 02-launch-ec2.sh         # Launch t2.micro, install Docker/Nginx via user-data
│   ├── 03-configure-affine.sh   # Deploy AFFiNE stack via SSH (runs locally)
│   ├── test-affine.sh           # Automated test suite (works with both v2 and v3)
│   └── 05-teardown.sh           # Tag-based sweep — destroys ALL project resources
├── config/                      # Git-ignored; holds bootstrap-outputs.env at runtime (v2 only)
├── cis/                         # CIS v2.0 security verification tools (Prowler, Lynis, docker-bench)
├── docs/                        # Deployment and CIS verification guides

```

## v3 — Terraform deployment (recommended)

### Prerequisites

- WSL (Ubuntu/Kali) or macOS/Linux
- AWS CLI configured (`aws sts get-caller-identity` works)
- [Terraform ≥ 1.5](https://developer.hashicorp.com/terraform/install)

### Deployment

```bash
git clone https://github.com/huanyan6/affine-aws-demo.git
cd affine-aws-demo/terraform

terraform init
cp terraform.tfvars.example terraform.tfvars   # edit if needed (region, domain, etc.)
terraform apply                                 # ~3 min to provision

# Monitor setup progress (AFFiNE takes ~10 min after apply):
$(terraform output -raw setup_log_command)

# Validate deployment:
cd .. && ./scripts/test-affine.sh --skip-tls-verify
```

When finished:
```bash
cd terraform && terraform destroy
```

### Key variables (`terraform.tfvars`)

| Variable | Default | Notes |
|---|---|---|
| `aws_region` | `ap-southeast-2` | Change to your preferred region |
| `affine_domain` | `""` | Set for Let's Encrypt HTTPS; leave empty for self-signed cert |
| `ssh_allowed_cidr` | auto-detected | Auto-fetches your public IP; override if needed |
| `ec2_instance_type` | `t2.micro` | Free Tier eligible |
| `local_ssh_key_path` | `~/.ssh/affine-demo-key.pem` | Where to write the generated key |

### Key design decisions (Terraform)

#### All secrets generated by Terraform
`random_password` resources produce the PostgreSQL password and AFFiNE secret. No manual `openssl rand` needed. Values are stored in Terraform state (git-ignored).

#### SSH key generated by Terraform
`tls_private_key` + `aws_key_pair` + `local_sensitive_file` replace the manual `aws ec2 create-key-pair` flow. The `.pem` is written to `~/.ssh/affine-demo-key.pem` with `0400` permissions automatically.

#### Deployer IP auto-detected
`data "http" "my_ip"` fetches `checkip.amazonaws.com` at plan time and restricts SSH to that CIDR. Override with `ssh_allowed_cidr` if your IP is dynamic or you need a different range.

#### Pure user-data, no SSH provisioner
All AFFiNE config (`.env`, `docker-compose.yml`, Nginx) is rendered via `templatefile()` at apply time with Terraform-substituted values, then runs as cloud-init on first boot. `terraform apply` finishes in ~3 min; AFFiNE is ready ~10 min later. Monitor via `setup_log_command` output.

#### Destroy replaces script 05
`terraform destroy` removes all resources tracked in state. Unlike the tag-based shell script approach, it precisely removes what this apply created — no more, no less.

### Terraform outputs

| Output | Description |
|---|---|
| `public_ip` | Elastic IP of the server |
| `access_url` | `https://` URL for AFFiNE |
| `ssh_command` | Ready-to-run SSH command |
| `setup_log_command` | SSH command to tail `/var/log/affine-setup.log` |
| `s3_bucket` | S3 bucket name |
| `ec2_user` | EC2 OS username |
| `ssh_key_path` | Local path to the SSH private key |

## v2 — Shell script deployment (legacy)

### Prerequisites

- WSL (Ubuntu/Kali) or macOS/Linux
- AWS CLI configured
- `jq`, `openssl`, `curl`, `ssh`, `scp` available in shell

### Deployment

```bash
git clone https://github.com/huanyan6/affine-aws-demo.git
cd affine-aws-demo
chmod +x scripts/*.sh

./scripts/01-aws-bootstrap.sh    # ~2 min  — prompts for local + EC2 usernames once
./scripts/02-launch-ec2.sh       # ~5 min  — waits for instance status checks
./scripts/03-configure-affine.sh  # ~10 min — waits for user-data, deploys via SSH
./scripts/test-affine.sh         # ~1 min  — validates deployment
```

When finished:
```bash
./scripts/05-teardown.sh
```

### Key design decisions (shell scripts)

#### Usernames set once in script 01
`01-aws-bootstrap.sh` prompts for local username and EC2 OS username; both are saved to `config/bootstrap-outputs.env` and reused by all subsequent scripts.

#### Script 03 runs locally, not on EC2
Orchestrates via SSH: polls until SSH is ready, waits for user-data to finish, SCPs files to `~/` then `sudo mv` to `/opt/affine/`, writes `.env` and `docker-compose.yml`, starts the stack.

#### Script 05 uses tag-based discovery
Queries AWS by tag/name pattern so all duplicates from repeated runs are cleaned up, not just the last recorded set.

## AFFiNE Docker image

Uses the official public image: `ghcr.io/toeverything/affine:stable`
- `ghcr.io/toeverything/affine-graphql:stable` is now private — do not use
- The `affine_migration` service runs DB migrations before the main container starts

## Common issues and fixes

| Symptom | Fix |
|---|---|
| `InvalidKeyPair.NotFound` on script 02 | Teardown deleted the AWS key but local `.pem` remained — script 01 detects and recreates both |
| `scp: Permission denied` on `/opt/affine/` | Script 03 SCPs to `~/` and `sudo mv`s — avoids running before user-data finishes |
| `curl` conflicts on `dnf install` | AL2023 ships `curl-minimal`; neither script 02 nor user_data.sh.tpl installs full `curl` |
| `unauthorized` / `denied` on `ghcr.io` | Image moved — use `ghcr.io/toeverything/affine:stable` |
| SSH post-quantum warning | `LogLevel=ERROR` in SSH_OPTS (scripts) / not an issue in Terraform (no SSH provisioner) |
| Terraform state lost | State is local by default; add an S3 backend block to `versions.tf` for team/persistent use |
